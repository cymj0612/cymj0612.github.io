CentOS7下安装hadoop3.0.3

发表于 2018-07-18 |

在安装hadoop之前，请自行学习安装虚拟机以及CentOS7系统。

一、安装JDK

1.centos下默认安装openjdk，需要先卸载。检查是否含有openjdk安装包，在终端窗口输入命令行：

java -version
rpm -qa | grep java

2.卸载openjdk：

rpm -e --nodeps + 需要卸载的文件名（注意，noarch文件不必删除）

例如：

rpm -e --nodeps java-1.7.0-openjdk-1.7.0.111-2.6.7.8.el7.x86_64
rpm -e --nodeps java-1.8.0-openjdk-1.8.0.102-4.b14.el7.x86_64
rpm -e --nodeps java-1.8.0-openjdk-headless-1.8.0.102-4.b14.el7.x86_64
rpm -e --nodeps java-1.7.0-openjdk-headless-1.7.0.111-2.6.7.8.el7.x86_64

执行完上述操作后，输入 java -version 查看是否卸载成功。

3.下载稳定版JDK文件，根据系统的版本选择合适的.tar.gz文件。在虚拟机home文件夹下新建文件hadoop，并将下载好的JDK文件放入hadoop文件夹中，解压：

tar -zxvf jdk-linux-x64.tar.gz(此处为你自己下载的版本)

4.安装完成后，配置环境变量，编辑/etc/profile文件：

vi /etc/profile/

按 i 进入文件插入模式，在文末最后空白处添加：

#java environment
export JAVA_HOME=/usr/java/jdk1.8.0_144（此处为jdk路径）
export CLASSPATH=.:${JAVA_HOME}/jre/lib/rt.jar:${JAVA_HOME}/lib/dt.jar:${JAVA_HOME}/lib/tools.jar
export PATH=$PATH:${JAVA_HOME}/bin

按 esc 键退出编辑模式，：wq 保存并退出文件。

source /etc/profile （让设置的环境变量生效）
oche $JAVA_HOME （查看jdk安装路径是否保存）

5.检查是否成功安装：

java -version

6.补充：

终端命令行输入是用户身份还是root身份，视情况而定，建议涉及到文件编辑等，以用户身份，如果提示权限要求，则以root 身份。
用户身份与root身份切换：

su root(用户切换至root)
su 用户名（即系统锁屏休眠的名字）（root切换至用户）

二、SSH免密设置

1.切换至root 身份，测试是否能免密登录：

ssh localhost

2.生成密钥：

ssh-keygen -t rsa    (根据提示操作，但凡有输入，全部回车键)

3.将密钥复制到目标服务器，即终端命令行前的root@目标服务器：

ssh-copy-ip +目标服务器  （例如： ssh-copy-ip xt）

4.测试，不报错，也不需要输入密码：

ssh +目标服务器  
#Last login:Thu Jul 19 09:34:27 2018(运行结果)

如此就配置好了。

三、安装Hadoop

1.在hadoop官网选择版本并下载，即Tarball下binary里，选择HTTP最后一个镜像文件（.tar.gz）,并放至hadoop文件夹中，切换至用户身份解压：

tar -zxvf hadoop-3.0.3.tar.gz(对应的版本)

2.检查hadoop是否成功安装：

cd /home/admin/hadoop/hadoop-3.0.3/bin/(进入hadoop安装目录,即解压路径)
hadoop version

此时，表明已经成功安装hadoop，如果不是，重启系统。

3.hadoop环境配置

vi /etc/profile/

按 i 进入文件插入模式，在文末最后空白处添加：

export HADOOP_HOME=/home/admin/hadoop/hadoop-3.0.3 (hadoop安装路径)
export PATH=$PATH:${HADOOP_HOME}/bin

按 esc 键退出编辑模式，：wq 保存并退出文件。

source /etc/profile （让设置的环境变量生效）
oche $HADOOP_HOME （查看hadoop安装路径是否保存）

4.jdk在hadoop中的环境配置：

cd /home/admin/hadoop/hadoop-3.0.3/etc/hadoop/
ls  (查看该文件目录下的文件)
vi hadoop-env.sh  (添加JAVA_HOME路径：JAVA_HOME=/usr/java/jdk1.8.0_144)
vi yarn-env.sh  (同上)

四、搭建hadoop伪分布模式

1.进入文件配置目录：

cd /home/admin/hadoop/hadoop-3.0.3/etc/hadoop/
ls

2.配置core-site.xml文件：

在hadoop-3.0.3目录下，创建文件夹tmp：

mkdir tmp
pwd     (获取tmp的路径，后面配置tmp)
vi core-site.xml

// 在 标签里面加上以下代码

<!--配置HDFS的NameNode-->  
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://localhost：9000</value> 
</property>
<!--配置DataNode保存数据的位置-->
<property>
    <name>hadoop.tmp.dir</name> 
    <value>/home/admin/hadoop/hadoop-3.0.3/tmp</value> 
</property>

3.配置hdfs-site.xml 文件：

vi hdfs-site.xml

// 在 标签里面加上以下代码

<!--配置HDFS的冗余度-->
<property>
    <name>dfs.replication</name> 
    <value>1</value> 
</property>  
<!--配置是否检查权限-->
<property>
    <name>dfs.permissions</name> 
    <value>false</value> 
</property>

4.配置 mapred-site.xml文件：

vi mapred-site.xml

// 在 标签里面加上以下代码

<!--配置MapReduce的框架-->
<property>
     <name>mapreduce.framework.name</name>
     <value>yarn</value>
</property>

5.配置 yarn-site.xml 文件：

vi yarn-site.xml

// 在 标签里面加上以下代码

<!--配置主节点ResourceManager的地址-->
<property>
    <name>yarn.resourcemanager.hostname</name>
    <value>localhost</value>
</property>
<!--配置节点NodeManager执行任务的方式-->
<property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
</property>

6.切换至root身份，格式化NameNode:

cd /home/admin/hadoop/hadoop-3.0.3/bin/
hdfs namenode -format

成功的话，会看到 “successfully formatted” 和 “Exitting with status 0” 的提示，若为 “Exitting with status 1” 则是出错　

7.启动hadoop：

cd /home/admin/hadoop/hadoop-3.0.3/
sbin/start-all.sh

如果出现下面情况，是因为缺少用户定义：

解决方案：

1.配置start-dfs.sh 和 stop-dfs.sh：

vi start-dfs.sh   （vi stop-dfs.sh）

//把以下代码复制到文件的空白处 ：wq 保存退出

HDFS_DATANODE_USER=root
HDFS_DATANODE_SECURE_USER=hdfs
HDFS_NAMENODE_USER=root
HDFS_SECONDARYNAMENODE_USER=root

2.配置 start-yarn.sh 和 stop-yarn.sh

vi start-yarn.sh  (vi stop-yarn.sh)

//把以下代码复制到文件的空白处 ：wq 保存退出

YARN_RESOURCEMANAGER_USER=root
HADOOP_SECURE_DN_USER=yarn
YARN_NODEMANAGER_USER=root

配置完成后，重新启动，正确情况如下图所示。

7.查看进程：

jps


红框内5个进程缺一不可，如果没有有一个没有启动，必须关闭所有所有进程（stop-all.sh），重新启动。
